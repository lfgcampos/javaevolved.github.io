Record-based error responses

Use records for concise, immutable error response types.

Code Comparison
‚úï Java 8 
// Verbose error class
public class ErrorResponse {
    private final int code;
    private final String message;
    // constructor, getters, equals,
    // hashCode, toString...
}
‚úì Java 16+ 
public record ApiError(
    int code,
    String message,
    Instant timestamp
) {
    public ApiError(int code, String msg) {
        this(code, msg, Instant.now());
    }
}

How it works

Records are perfect for error responses ‚Äî they're immutable, have built-in equals/hashCode for comparison, and toString for logging. Custom constructors add validation or defaults.
