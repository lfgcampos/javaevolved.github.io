## Architecture

This site uses a **JSON/YAML-first** build pipeline:

- **Source of truth**: Individual `content/category/slug.json` files (112 patterns across 11 category folders)
- **Templates**: `templates/` â€” shared HTML templates with `{{placeholder}}` tokens for content and UI strings
- **Generator**: `html-generators/generate.java` â€” JBang script that produces all HTML pages, localized variants, and `data/snippets.json`
- **Translations**: `translations/strings/{locale}.yaml` for UI strings, `translations/content/{locale}/` for pattern content (YAML)
- **Deploy**: GitHub Actions runs the generator and deploys to GitHub Pages

Generated files (`site/category/*.html`, `site/{locale}/`, and `site/data/snippets.json`) are in `.gitignore` â€” never edit them directly.

## Build & run locally

### Prerequisites

- **Java 25+** (e.g. [Temurin](https://adoptium.net/))

### Generate and serve

```bash
# Generate all HTML pages and data/snippets.json into site/
jbang html-generators/generate.java

# Serve locally
jwebserver -b 0.0.0.0 -d site -p 8090
# Open http://localhost:8090
```

The fat JAR is a self-contained ~2.2 MB file with all dependencies bundled. [JBang](https://jbang.dev) is needed to run the generator.

For development on the generator itself, you can use JBang or Python â€” see [html-generators/README.md](html-generators/README.md) for details.
