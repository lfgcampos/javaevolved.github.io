Virtual thread executor

Use virtual thread executors for unlimited lightweight concurrency.

Code Comparison
‚úï Java 8

    Executors.newFixedThreadPool(10);
try {
    futures = tasks.stream()
        .map(t -> exec.submit(t))
        .toList();

    exec.shutdown();
}
‚úì Java 21+

    var futures = tasks.stream()
        .map(exec::submit)
        .toList();
}

How it works

The virtual thread executor creates a new virtual thread for each task. No pool sizing needed ‚Äî virtual threads are cheap enough to create millions of them.
